# Конспект презентации

## 1. Идея проекта

Мы сделали интерактивный анализатор записи жестов в системе Шрифт РЖЯ.

Важно: это не переводчик жестового языка на русский. Программа не должна угадывать русское слово. Её задача — показать, из каких фонетико-графических компонентов состоит запись жеста.

Пример:

```text
(ЁЁ)*.Эо
```

Разбор:

```text
локализация: (ЁЁ) — висок
контакт: * — коснуться
рука:
  ориентация: .Э
  конфигурация: о
```

## 2. Кому полезно

- студентам, которые учатся читать записи в Шрифте РЖЯ;
- преподавателям как демонстрационный инструмент;
- исследователям, которым нужно быстро проверить структуру записи;
- людям, которые набирают записи вручную и хотят видеть ошибки.

## 3. Почему задача не тривиальная

Кажется, что можно просто сделать таблицу символов. Но в системе много значений задаётся цепочками:

```text
.Э      ориентация
(ЁЁ)    локализация на голове
9Ё10    локализация на кисти
9вЬ0    относительная локализация
:%()    немануальный компонент
ЯЯ      движение неведущей руки
```

Если читать посимвольно, разбор будет неправильным. Поэтому нужен parser, который сначала ищет самые длинные известные цепочки.

## 4. Технические задачи

1. Собрать справочник базовых компонентов из документации.
2. Реализовать нормализацию ввода.
3. Сделать токенизатор longest-match-first.
4. Сгруппировать токены в кадры и таймлайны.
5. Выделить внутри кадра руки, локализации и контакты.
6. Добавить валидатор нестандартных случаев.
7. Сделать интерфейс: подсветка, дерево, таблица, JSON.
8. Написать README и тесты.

## 5. Лингвистические задачи

- Разделить уровни записи: конфигурация, ориентация, локализация, движение, контакт, немануальные компоненты.
- Учесть, что запись жеста линейная, но описывает пространственную структуру.
- Различать кадры и таймлайны.
- Не обещать машинный перевод, потому что значение целого жеста требует отдельного словаря.
- Обрабатывать сокращения и неоднозначности.

## 6. Сложности

### Неоднозначные символы

`!` может быть контактом «ударить» или модификатором «выразительно».

`;` может быть контактом «тереть» или модификатором «пауза».

Решение: интерпретировать по контексту и показывать информационное сообщение.

### Составные цепочки

`9!0` — не локализация, а быстрое движение. Поэтому специальные цепочки должны распознаваться раньше общих групп `9...0`.

### Неполные записи

Например:

```text
(ЁЁ)*.Э
```

Тут есть ориентация, но нет конфигурации руки. Программа не падает, а предупреждает.

### Нераспознанные группы

Если пользователь вводит:

```text
9abc0
```

программа сохраняет это как один токен «нераспознанная группа», а не ломает строку на неизвестные символы.

## 8. Как проект соответствует критериям

- Репозиторий полный: есть `README.md`, исходный код, тесты, документация.
- Код прокомментирован JSDoc-комментариями.
- Работоспособность проверяется через `npm test`.
- Робастность: неизвестные символы, неполные ориентации и неизвестные группы не ломают приложение.
- Расширенный функционал: подсветка, дерево, валидатор, JSON-экспорт, мини-словарь, примеры.

## 9. Бонусная формулировка для финала

Мы сделали не таблицу символов и не переводчик, а маленький «компилятор» для записи Шрифта РЖЯ: он превращает линейную строку в структурное дерево жеста и объясняет, как устроена запись.
